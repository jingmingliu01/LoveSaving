# LoveSaving Firebase DB Spec (Current)

Last updated: 2026-02-26  
Applies to project: `lovesaving-72814`

## 1. Scope
This spec describes the **currently deployed** backend model for:
- Firestore data model
- Firebase Storage media layout
- Callable Cloud Function used for user resolution

Source of truth:
- `../firestore.rules`
- `../storage.rules`
- `../functions/index.js`

## 2. Core invariants
- A group has exactly two members (`memberIds.size() == 2`).
- Group data is readable by participants even when group status is `inactive`.
- Group writes are allowed only while status is `active` (except the controlled soft-unlink transition).
- System timestamps are server-time guarded in rules (`field == request.time`).
- Media binary is stored in Storage, not inline in Firestore.

## 3. Firestore model

### 3.1 `users/{uid}`
Purpose: auth-adjacent profile and app state.

Fields:
- `displayName: string`
- `email: string`
- `emailLower: string`
- `photoURL: string|null` (optional)
- `currentGroupId: string|null` (optional)
- `fcmToken: string|null` (optional)
- `createdAt: timestamp`
- `updatedAt: timestamp`

Update policy:
- Self can update only: `displayName`, `photoURL`, `fcmToken`, `updatedAt`.
- `createdAt`, `email`, `emailLower`, `currentGroupId` are immutable in self-update path.
- `currentGroupId` is changed through link/unlink state-machine paths.

### 3.2 `invites/{inviteId}`
Purpose: two-user linking lifecycle.

Fields:
- `fromUid: string`
- `fromDisplayName: string|null` (optional)
- `fromEmail: string|null` (optional)
- `toUid: string`
- `status: "pending" | "accepted" | "rejected" | "expired"`
- `createdAt: timestamp`
- `respondedAt: timestamp|null` (optional)
- `expiresAt: timestamp|null` (optional)

Transitions:
- Create only as `pending`.
- Update affects only `status`, `respondedAt`.
- `accepted/rejected` can be set by `toUid`.
- `expired` can be set by invite participants.

### 3.3 `groups/{groupId}`
Purpose: shared access boundary and aggregate root.

Fields:
- `groupName: string`
- `memberIds: string[2]`
- `createdBy: string`
- `sourceInviteId: string`
- `status: "active" | "inactive"`
- `loveBalance: int`
- `lastEventAt: timestamp`
- `createdAt: timestamp`
- `updatedAt: timestamp`

Important constraints:
- `createdBy` must be in `memberIds`.
- Create requires accepted invite consistency:
  - caller is invite `toUid`
  - `createdBy` equals invite `fromUid`

### 3.4 `groups/{groupId}/events/{eventId}`
Purpose: immutable core event log + limited editable user fields.

Fields:
- `createdBy: string`
- `type: "deposit" | "withdraw"`
- `tapCount: int` (`> 0`)
- `delta: int`
- `note: string|null` (optional)
- `occurredAt: timestamp` (business time)
- `location: map`
  - `lat: int|float`
  - `lng: int|float`
  - `addressText: string|null` (optional)
- `media: list<map>|null` (optional, max 3)
  - `storagePath: string` (must match `groups/{groupId}/events/{eventId}/{filename}`)
  - `contentType: image/(jpeg|jpg|png|heic|heif|webp)`
- `createdAt: timestamp`
- `recordedAt: timestamp` (server record time)
- `updatedAt: timestamp`

Update policy:
- Editable only by event creator while group is active.
- Mutable keys only: `note`, `location`, `media`, `updatedAt`.
- Immutable: `createdBy`, `type`, `tapCount`, `delta`, `occurredAt`, `createdAt`, `recordedAt`.

### 3.5 `groups/{groupId}/chats/{chatId}`
Fields:
- `createdBy: string`
- `title: string|null` (optional)
- `createdAt: timestamp`
- `updatedAt: timestamp`
- `lastMessageAt: timestamp`

Update policy:
- Mutable keys: `title`, `updatedAt`, `lastMessageAt`
- `lastMessageAt` must be monotonic non-decreasing.

### 3.6 `groups/{groupId}/chats/{chatId}/messages/{messageId}`
Fields:
- `role: "user" | "assistant" | "system"`
- `content: string`
- `createdAt: timestamp`

Policy:
- Create-only (no update/delete).

### 3.7 Deprecated path
`userLookup/{uid}` is disabled in rules (`read/write: false`) and no longer used by app runtime.

## 4. Storage model

Path:
- `groups/{groupId}/events/{eventId}/{filename}`

Rules summary:
- Read: group participant.
- Write: active group member + valid image upload.
- Upload constraints:
  - `request.resource != null`
  - `size <= 8MB`
  - content type allow-list: `jpeg/jpg/png/heic/heif/webp`

Note:
- Storage rules use `firestore.get(...)` to validate membership and group status.

## 5. Callable function

`resolveUserIdentifier`:
- Callable, auth required.
- Input: `identifier` string.
- Behavior:
  - If contains `@`, resolve by email via Admin SDK.
  - Else resolve by UID via Admin SDK.
- Output:
  - `uid`
  - `displayName|null`
  - `email|null`

## 6. Time semantics
- `occurredAt`: business event time (client meaning).
- `recordedAt`: server recording time (`request.time` guarded).
- `createdAt/updatedAt`: server-time guarded for rule consistency and auditability.
