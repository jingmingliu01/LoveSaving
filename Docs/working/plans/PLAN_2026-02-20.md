## Scope
- In:
  - iOS app (SwiftUI)
  - Firebase Auth, Firestore, Storage, FCM integration
  - Core UI/flows from wireframe
  - Security rules, tests, and acceptance validation
- Out:
  - LLM Insight chat and chat history
  - Admin tooling
  - Localization and advanced visual polish

## Public APIs / Interfaces / Types
- Firestore collections:
  - `users`
  - `invites`
  - `groups`
  - `groups/{groupId}/events`
- Service interfaces:
  - `AuthService`
  - `InviteService`
  - `GroupService`
  - `EventService`
  - `MediaService`
  - `NotificationService`
- Critical write path:
  - `createEventAndUpdateGroup(eventInput)` with transaction semantics:
    - `loveBalance += delta`
    - `lastEventAt = max(lastEventAt, occurredAt)`
    - `updatedAt = now`
- Unlink interface:
  - `softUnlink(groupId, memberIds)`:
    - set group `status = "inactive"`
    - clear both users’ `currentGroupId`

## Test Cases and Acceptance Scenarios
[ ] Auth scenarios: register/login/logout/password change, including error states.  
[ ] Invite scenarios: send/receive/accept/reject/expire and correct group/user state transitions.  
[ ] Home scenarios: multiple taps generate one event write with correct `tapCount` and `delta`.  
[ ] Concurrency scenario: both users write events at the same time without `loveBalance` drift.  
[ ] Journey scenarios: list ordering by `occurredAt desc` and map/list data consistency.  
[ ] Media scenarios: upload success, upload failure fallback, and event record durability.  
[ ] Security scenarios: unauthorized user cannot read/write group data or unrelated invites.  
[ ] Unlink scenario: group set to inactive, users detached, historical data preserved.  
[ ] Notification scenarios: permission prompt, token handling, local reminder trigger.  

## Assumptions and Defaults
- Phase 1 is MVP-only (no LLM Insight execution yet).  
- Stack defaults to SwiftUI + async/await with iOS 17 baseline.  
- Invite correctness is UID-based; email can be used only as a lookup/entry method.  
- Debounce window defaults to `1.5s`; empty note defaults to time + location string.
