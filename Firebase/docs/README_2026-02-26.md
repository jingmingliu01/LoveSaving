# Firebase Setup Notes

This `docs/` directory describes the deployable Firebase configuration under `Firebase/`.

- Firebase project id: `lovesaving-72814`

## Current architecture (active)
- Firestore rules: `firestore.rules`
- Firestore indexes: `firestore.indexes.json`
- Storage rules: `storage.rules`
- Cloud Functions (callable resolver): `functions/index.js`
- Firebase project config: `firebase.json`

## No inline media fallback
The app now uses **Firebase Storage** for event media.
- Media is uploaded to Storage path: `groups/{groupId}/events/{eventId}/{filename}`
- Firestore `events.media[]` stores metadata (`storagePath`, `contentType`)
- Inline data-URI media in Firestore is no longer used in live runtime

## Deploy (Firebase CLI)
Run from the `Firebase/` directory (one level above this file):

```bash
firebase login
firebase deploy --only functions,firestore:rules,firestore:indexes,storage --project lovesaving-72814
```

### First-time Storage setup
If Storage has never been initialized in console, open:
- https://console.firebase.google.com/project/lovesaving-72814/storage
- Click **Get Started** once

When deploying Storage rules that reference Firestore (`firestore.get(...)`), CLI may prompt to grant an IAM role for cross-service rule evaluation. Accept it.
